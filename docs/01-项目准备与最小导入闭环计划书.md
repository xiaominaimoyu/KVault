# 阶段 1：项目准备与最小导入闭环计划书

> 对应目标：先建立可运行项目骨架，并跑通“选择文件 → 解析 → 切分 → 入库”的最小闭环。
> 建议周期：第 1 周
> 阶段性质：MVP 基础阶段

---

## 1. 阶段目标

本阶段不追求完整功能和界面美化，只验证系统的核心路径是否成立。

完成后，项目应具备一个最小可运行桌面应用：用户可以启动 PySide6 主窗口，通过导入入口选择 TXT、Markdown、PDF 文件，系统将文件复制到本地数据目录，解析出文本，切分为 chunk，并写入 ChromaDB。

---

## 2. 本阶段范围

### 2.1 必须完成

- 初始化 Python 项目目录结构。
- 创建虚拟环境与依赖清单。
- 创建基础配置文件 `config.json.example`。
- 搭建 PySide6 主窗口三栏布局骨架。
- 实现基础文档解析器，优先支持 `.txt`、`.md`、`.pdf`。
- 实现文本切分模块。
- 实现 ChromaDB 向量库封装的最小写入能力。
- 实现 GUI 文件选择导入入口。
- 跑通导入、解析、切分、入库流程。

### 2.2 暂不完成

- 不做完整 SQLite 元数据管理。
- 不做真实标签、分区、批量管理。
- 不做完整语义检索 UI。
- 不做 MCP。
- 不做打包。
- 不做复杂 PDF OCR 和 Docling 解析。

---

---

## 4. 任务拆解

### 4.1 环境初始化

创建 Python 3.10+ 虚拟环境，安装第一阶段必要依赖：`PySide6`、`PyMuPDF`、`langchain`、`langchain-community`、`chromadb`、`numpy`。

输出物：

- `requirements.txt`
- 可正常启动的虚拟环境
- 可运行的空白应用入口

验证方式：

```bash
python main.py
```

能看到 PySide6 主窗口即通过。

### 4.2 配置文件与路径管理

实现 `core/config.py`，负责加载默认配置、创建数据目录、提供统一路径。

最低配置项包括：

- `data.files_dir`
- `data.chroma_db_dir`
- `chunking.chunk_size`
- `chunking.chunk_overlap`

验收标准：

- 首次启动时自动创建 `data/files/` 和 `data/chroma_db/`。
- 配置缺失时使用默认值。

### 4.3 GUI 主窗口骨架

实现 `gui/main_window.py`。

界面先保持简单，包含：

- 顶部导入按钮。
- 左侧占位导航栏。
- 中间文档列表占位区。
- 右侧预览占位区。
- 底部状态栏。

验收标准：

- 窗口尺寸默认 1280x800。
- 点击导入按钮能打开文件选择框。
- 状态栏能显示导入过程中的简短信息。

### 4.4 文档解析模块

实现 `core/document_parser.py`。

第一阶段支持：

- `.txt`：按 UTF-8 读取，必要时尝试系统默认编码兜底。
- `.md`：按纯文本读取。
- `.pdf`：使用 PyMuPDF 提取每页文本。

建议输出统一数据结构：

```python
ParsedDocument(
    content: str,
    metadata: dict
)
```

验收标准：

- 传入一个 TXT 文件能返回正文。
- 传入一个 Markdown 文件能返回正文。
- 传入一个普通文本型 PDF 能返回页面文本。
- 不支持的格式返回清晰错误。

### 4.5 文本切分模块

实现 `core/text_splitter.py`。

默认参数：

- `chunk_size = 500`
- `chunk_overlap = 100`
- `separators = ["\n\n", "\n", "。", "！", "？", " ", ""]`

验收标准：

- 输入长文本后返回 chunk 列表。
- 每个 chunk 包含 `chunk_index`、`total_chunks`、`source_file`。
- 空文本不会导致程序崩溃。

### 4.6 ChromaDB 最小封装

实现 `core/vector_store.py`。

本阶段可以先使用临时向量或固定维度测试向量，以验证 ChromaDB 写入路径。若 Ollama 已可用，也可以直接接入真实 Embedding，但不强制。

最低能力：

- 初始化持久化 collection。
- 批量写入 chunk 文本、metadata、embedding。
- 获取 collection 统计数量。

验收标准：

- 导入后 ChromaDB collection 数量增加。
- 重启应用后数据仍存在。

### 4.7 最小导入流程

在 GUI 中串联：

```text
选择文件 → 复制到 data/files/ → 解析 → 切分 → 写入 ChromaDB → 状态栏提示完成
```

验收标准：

- 导入一个 `.txt` 成功。
- 导入一个 `.md` 成功。
- 导入一个 `.pdf` 成功。
- 导入失败时 GUI 不崩溃，并显示失败原因。

---

## 6. 风险与控制

| 风险 | 处理方式 |
|------|----------|
| PySide6 安装或运行异常 | 先验证空窗口，避免业务代码写完后才发现环境问题 |
| PDF 解析结果为空 | 本阶段只支持文本型 PDF，扫描件 OCR 后续阶段再做 |
| ChromaDB 与依赖版本冲突 | 先锁定一个可运行版本，后续统一升级 |
| Embedding 模型未准备好 | 本阶段允许使用测试向量，真实 Embedding 放到阶段 2 |

---

## 7. 完成定义

当一个普通用户可以启动应用，选择一个 TXT、Markdown 或 PDF 文件，并看到系统完成导入入库提示时，本阶段完成。
