# 阶段 4：GUI 管理功能完善计划书

> 对应目标：把桌面端从“可测试工具”完善为“可日常使用的知识库管理器”。
> 建议周期：第 4 周前半
> 阶段性质：桌面体验与管理能力阶段

---

## 1. 阶段目标

本阶段重点补齐 GUI 管理能力。完成后，用户应能在界面中完成分区管理、标签管理、文档移动、删除、重新索引、状态查看和设置调整。

这一阶段不新增大规模后端能力，主要把前几个阶段已有的数据能力组织成顺手的桌面操作。

---

### 2.1 必须完成

- 完善标签云筛选。

## 4. 任务拆解

### 4.1 左侧分区树

功能：

- 显示全部文档。
- 显示默认分区。
- 显示每个分区文档数量。
- 点击分区过滤文档列表。
- 右键新建、重命名、删除分区。

删除分区策略：

- 第一版不物理删除文档。
- 删除分区时，将其中的文档移动到默认分区或“未分类”。

验收标准：

- 分区增删改后 SQLite 同步更新。
- 文档列表过滤结果正确。
- 空分区也能显示。

### 4.2 标签云

功能：

- 显示常用标签。
- 点击标签过滤文档列表。
- 支持取消筛选。
- 文档详情中可编辑标签。

验收标准：

- 标签来自 SQLite，而不是写死。
- 一个文档可拥有多个标签。
- 标签筛选与分区筛选可组合。

### 4.3 向量库状态面板

显示信息：

- 总文档数。
- 已索引文档数。
- 失败文档数。
- 总 chunk 数。
- 当前 Embedding 模型。
- ChromaDB 路径。

验收标准：

- 导入、删除、重新索引后状态面板刷新。
- 状态数据来自真实存储。

### 4.4 文档右键菜单

菜单项：

- 打开原始文件。
- 删除文档。
- 移动到分区。
- 编辑标签。
- 重新索引。
- 查看错误详情。

验收标准：

- 只对选中文档生效。
- 批量选择时支持批量删除、批量移动分区。
- 失败文档可以查看错误原因。

### 4.5 删除文档

删除需要同步处理：

```text
删除 ChromaDB 中该文档的 chunks
→ 删除 SQLite chunk 记录
→ 删除 SQLite document 记录
→ 可选删除 data/files/ 中副本
```

第一版建议弹窗确认是否删除原始副本。

验收标准：

- 删除后文档列表消失。
- 删除后检索不到该文档内容。
- 删除失败时能提示具体原因。

### 4.6 重新索引

重新索引流程：

```text
删除旧 chunk 和旧向量
→ 保留文档记录
→ 重新解析 stored_path
→ 重新切分
→ 重新生成 embedding
→ 写入新 chunk 和向量
→ 更新状态
```

验收标准：

- 重新索引不会生成重复结果。
- 失败时保留错误状态。

### 4.7 设置对话框

设置项：

- 数据目录。
- Embedding provider。
- Ollama base URL。
- Embedding 模型名。
- chunk size。
- chunk overlap。
- default top_k。
- similarity threshold。
- MCP 开关。
- 主题模式。

注意：

Embedding 模型和切分参数变更后，只影响后续导入。已有文档需要用户手动重新索引。

验收标准：

- 设置保存到配置文件。
- 非法配置有校验提示。
- 修改后界面状态栏能反映关键配置。

### 4.8 错误提示与日志

新增日志文件：

```text
data/logs/app.log
```

记录：

- 应用启动。
- 导入成功/失败。
- Embedding 调用失败。
- ChromaDB 写入失败。
- MCP 启动失败。

验收标准：

- GUI 错误提示简短友好。
- 详细异常写入日志。
- 用户可以从设置或菜单打开日志路径。

## 6. 风险与控制

| 风险 | 处理方式 |
|------|----------|
| GUI 功能过多导致代码混乱 | 每个面板保持单一职责，业务逻辑放在 core 层 |
| 删除操作误删文件 | 所有删除都先确认，默认只删索引和副本，不碰原始路径 |
| 重新索引产生重复向量 | 重建前必须按 document_id 删除旧向量 |
| 设置变更影响旧数据 | 明确提示“仅影响后续导入，旧文档需重新索引” |

---

## 7. 完成定义

当用户可以完全通过 GUI 管理文档、分区、标签、索引状态和关键配置时，本阶段完成。
