# 阶段 3：检索体验与格式扩展计划书

> 对应目标：让知识库真正可用，支持语义检索、结果预览和更多文档格式。
> 建议周期：第 3 周
> 阶段性质：核心使用体验阶段

---

## 1. 阶段目标

本阶段重点是“搜得到、看得懂、能判断结果是否可信”。完成后，用户可以在 GUI 中输入自然语言问题，系统返回相关文本片段、相似度、来源文档和 chunk 位置。

同时，本阶段补齐 DOCX、XLSX、PPTX 三类常见办公文档解析能力。

---

## 3. 检索服务设计

新增 `core/retriever.py`，组合：

- `EmbeddingService`
- `VectorStore`
- `MetadataManager`

核心流程：

```text
query 文本
→ embed_query()
→ vector_store.search()
→ 合并 SQLite 文档元信息
→ 过滤低于阈值的结果
→ 返回 SearchResult 列表
```

建议 `SearchResult` 字段：

| 字段 | 说明 |
|------|------|
| `chunk_id` | 文本块 ID |
| `document_id` | 文档 ID |
| `document_name` | 来源文档名 |
| `content` | 命中的文本片段 |
| `score` | 相似度分数 |
| `chunk_index` | chunk 序号 |
| `metadata` | 额外元数据 |

## 4. 任务拆解

### 4.1 Retriever 实现

最低接口：

```python
class Retriever:
    def search(
        self,
        query: str,
        top_k: int = 5,
        filters: dict | None = None,
    ) -> list[SearchResult]:
        ...
```

验收标准：

- 空查询直接返回空列表。
- 正常查询返回按相似度排序的结果。
- 支持 `top_k`。
- 支持按分区和标签过滤的接口参数，即使 GUI 暂时不完整使用。

### 4.2 检索测试面板

完善 `gui/preview_panel.py`。

面板包含：

- 查询输入框。
- 检索按钮。
- Top-K 设置，第一版可固定为 5。
- 结果列表。
- 相似度颜色提示。

颜色建议：

- `score >= 0.8`：高相关。
- `0.5 <= score < 0.8`：中等相关。
- `score < 0.5`：弱相关。

验收标准：

- 输入查询后能异步返回结果。
- 查询期间 GUI 不假死。
- 无结果时显示明确空状态。

### 4.3 检索结果与预览联动

点击结果项时：

```text
选中来源文档
→ 加载文档预览
→ 滚动或高亮对应 chunk
```

第一版可以先做到“切换到对应文档并显示 chunk 内容”，不强制实现精确滚动。

验收标准：

- 用户可以从检索结果追溯到来源文档。
- 结果中必须显示文档名和 chunk 序号。

### 4.4 顶部全局搜索栏

全局搜索栏用于过滤文档列表，不等同于语义检索。

过滤字段：

- 文件名
- 扩展名
- 标签
- 分区

验收标准：

- 输入关键字后文档列表实时过滤。
- 清空输入后恢复完整列表。
- 过滤不触发向量检索。

### 4.5 DOCX 解析

使用 `python-docx`。

解析内容：

- 段落文本。
- 标题层级。
- 表格文本，按行拼接。

验收标准：

- 普通 Word 文档可导入。
- 标题和正文顺序基本保留。
- 空段落不过度污染文本。

### 4.6 XLSX 解析

使用 `openpyxl`。

解析策略：

- 按工作表读取。
- 每行拼接为一行文本。
- 保留 sheet 名称。
- 跳过完全空行。

验收标准：

- 多 Sheet 文件可导入。
- 检索结果能体现表格内容。

### 4.7 PPTX 解析

使用 `python-pptx`。

解析策略：

- 按幻灯片读取。
- 提取文本框内容。
- 保留页码。

验收标准：

- 普通 PPTX 可导入。
- 结果中能追溯到幻灯片页码或 chunk 序号。

### 4.8 检索参数调优

使用真实样本文档测试：

- chunk size：500
- overlap：100
- top_k：5
- threshold：0.5

验收标准：

- 至少准备 10 份测试文档。
- 用 10 个自然语言查询验证结果相关性。
- 记录明显失败案例，作为后续优化依据。

## 6. 风险与控制

| 风险 | 处理方式 |
|------|----------|
| 相似度分数不稳定 | 先记录样例，不急着引入 Rerank |
| XLSX 文本结构混乱 | 第一版以可检索为主，复杂表格语义后续优化 |
| PPTX 图文混排信息缺失 | 第一版只提取文本框内容 |
| 搜索栏和语义检索概念混淆 | UI 上区分“文档过滤”和“语义检索测试” |
